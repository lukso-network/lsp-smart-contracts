  <action>
Create the main abstract contract implementing the full AccessControlExtended logic.

**File: AccessControlExtendedAbstract.sol**
- `pragma solidity ^0.8.27;`
- Inherit: `abstract contract AccessControlExtendedAbstract is IAccessControlExtended, LSP7DigitalAsset`
- Import: LSP7DigitalAsset, IAccessControlExtended, IAccessControl (for interfaceId), IAccessControlEnumerable (for interfaceId), EnumerableSet, errors, constants

**Storage (composition, NOT inheriting OZ AccessControl):**
```solidity
using EnumerableSet for EnumerableSet.AddressSet;
using EnumerableSet for EnumerableSet.Bytes32Set;

bytes32 public constant DEFAULT_ADMIN_ROLE = 0x00;

// role admin mapping (which role administers which role)
mapping(bytes32 => bytes32) private _roleAdmins;

// Forward lookup: role -> members (replaces OZ AccessControlEnumerable._roleMembers)
mapping(bytes32 => EnumerableSet.AddressSet) private _roleMembers;

// Reverse lookup: address -> roles (our extension)
mapping(address => EnumerableSet.Bytes32Set) private _addressRoles;

// Auxiliary data: role -> address -> bytes (our extension)
mapping(bytes32 => mapping(address => bytes)) private _roleData;
```

**Constructor:**
```solidity
constructor(address newOwner_) {
    _grantRole(DEFAULT_ADMIN_ROLE, newOwner_);
}
```

**Public functions (OZ IAccessControl):**

- `hasRole(bytes32 role, address account)` -> `_roleMembers[role].contains(account)` (direct lookup only; owner implicit access handled in `_checkRole`, not `hasRole`)
- `getRoleAdmin(bytes32 role)` -> `_roleAdmins[role]` (returns DEFAULT_ADMIN_ROLE by default since mapping defaults to 0x00)
- `grantRole(bytes32 role, address account)` -> calls `_checkRole(getRoleAdmin(role))` then `_grantRole(role, account)`. This is the standard OZ guard pattern. Because `_checkRole` gives owner implicit pass, owner can always grant.
- `revokeRole(bytes32 role, address account)` -> calls `_checkRole(getRoleAdmin(role))` then `_revokeRole(role, account)`. Same guard pattern.
- `renounceRole(bytes32 role, address callerConfirmation)` -> requires `callerConfirmation == msg.sender` (OZ pattern to prevent accidental renounce). Calls `_revokeRole(role, msg.sender)`. Allow renouncing DEFAULT_ADMIN_ROLE (follow OZ standard; owner still has implicit admin via _checkRole, so this is safe).

**Public functions (OZ IAccessControlEnumerable):**

- `getRoleMember(bytes32 role, uint256 index)` -> `_roleMembers[role].at(index)`
- `getRoleMemberCount(bytes32 role)` -> `_roleMembers[role].length()`

**Public functions (IAccessControlExtended):**

- `rolesOf(address account)` -> `_addressRoles[account].values()`
- `grantRoleWithData(bytes32 role, address account, bytes calldata data)` -> calls `_checkRole(getRoleAdmin(role))`, then `_grantRole(role, account)`, then if `data.length > 0`, sets `_roleData[role][account] = data` and emits `RoleDataChanged`. If the role was already held (i.e., `_grantRole` was a no-op), still update data if provided and emit `RoleDataChanged`.
- `setRoleData(bytes32 role, address account, bytes calldata data)` -> calls `_checkRole(getRoleAdmin(role))` for admin authority check (owner/DEFAULT_ADMIN_ROLE/role's admin). Per CONTEXT.md, does NOT revert if account lacks the role (allows pre-configuration). Sets `_roleData[role][account] = data` and emits `RoleDataChanged`.
- `getRoleData(bytes32 role, address account)` -> returns `_roleData[role][account]`

**Internal functions:**

- `_grantRole(bytes32 role, address account)` internal virtual:
  - If `_roleMembers[role].add(account)` returns true (was not already a member):
    - `_addressRoles[account].add(role)`
    - `emit RoleGranted(role, account, msg.sender)`
  - (No-op if already holds role, matching OZ behavior)

- `_revokeRole(bytes32 role, address account)` internal virtual:
  - If `_roleMembers[role].remove(account)` returns true (was a member):
    - `_addressRoles[account].remove(role)`
    - `emit RoleRevoked(role, account, msg.sender)`
    - Auto-clear data (BASE-09): if `_roleData[role][account].length > 0`, delete it and `emit RoleDataChanged(role, account, "")`

- `_checkRole(bytes32 role)` internal view virtual:
  - Calls `_checkRole(role, msg.sender)`

- `_checkRole(bytes32 role, address account)` internal view virtual:
  - If `account == owner()` -> return (owner implicitly passes all checks, per locked CONTEXT.md decision)
  - If `!hasRole(role, account)` -> `revert AccessControlExtendedUnauthorized(account, role)`
  - This means owner bypasses ALL onlyRole checks (simplest, most consistent -- see RESEARCH.md Open Question 1 recommendation)

- `_setRoleAdmin(bytes32 role, bytes32 adminRole)` internal virtual:
  - Sets `_roleAdmins[role] = adminRole`
  - `emit RoleAdminChanged(role, previousAdmin, adminRole)` (get previousAdmin from `_roleAdmins[role]` before overwriting)
  - Available for extensions to use per locked decision on custom role admins

- `onlyRole(bytes32 role)` modifier:
  - Calls `_checkRole(role)` which calls `_checkRole(role, msg.sender)`

**Ownership sync (locked decision):**

Override `_transferOwnership(address newOwner)`:
```solidity
function _transferOwnership(address newOwner) internal virtual override {
    address oldOwner = owner();
    super._transferOwnership(newOwner);
    if (oldOwner != address(0)) {
        _revokeRole(DEFAULT_ADMIN_ROLE, oldOwner);
    }
    if (newOwner != address(0)) {
        _grantRole(DEFAULT_ADMIN_ROLE, newOwner);
    }
}
```
