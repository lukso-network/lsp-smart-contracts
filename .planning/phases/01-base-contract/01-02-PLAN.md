**Test contract setup:**
- `pragma solidity ^0.8.22;` (matching existing test pragma)
- `contract AccessControlExtendedTest is Test`
- Test addresses: `owner = address(this)`, `nonOwner = vm.addr(100)`, `account1 = vm.addr(101)`, `account2 = vm.addr(102)`, `account3 = vm.addr(103)`

**Test sections (use `// ============================================================` separators matching codebase style):**

**Section 1: Constructor initialization (TEST-01)**
- `test_ConstructorGrantsDefaultAdminRoleToOwner` -- hasRole(DEFAULT_ADMIN_ROLE, owner) is true
- `test_ConstructorOwnerVisibleInEnumeration` -- getRoleMember(DEFAULT_ADMIN_ROLE, 0) == owner, getRoleMemberCount(DEFAULT_ADMIN_ROLE) == 1
- `test_ConstructorOwnerVisibleInReverseLookup` -- rolesOf(owner) contains DEFAULT_ADMIN_ROLE

**Section 2: grantRole / revokeRole (TEST-01)**
- `test_OwnerCanGrantRole` -- grant TEST_ROLE to account1, verify hasRole, verify RoleGranted event emitted
- `test_DefaultAdminCanGrantRole` -- grant DEFAULT_ADMIN_ROLE to account1, then account1 grants TEST_ROLE to account2 (prank as account1)
- `test_OwnerImplicitAdminBypass` -- owner can grant any role without explicitly having DEFAULT_ADMIN_ROLE enumerated (owner has it from constructor, but test that _checkRole passes for owner even for custom admin roles)
- `test_NonAdminCannotGrantRole` -- prank as nonOwner, expect revert with AccessControlExtendedUnauthorized
- `test_GrantRoleEmitsEvent` -- vm.expectEmit for RoleGranted(role, account, sender)
- `test_GrantRoleIdempotent` -- granting same role twice is no-op (no duplicate event, member count stays 1)
- `test_OwnerCanRevokeRole` -- grant then revoke TEST_ROLE from account1, verify hasRole returns false
- `test_RevokeRoleEmitsEvent` -- vm.expectEmit for RoleRevoked(role, account, sender)
- `test_RevokeRoleIdempotent` -- revoking non-held role is no-op (no event)
- `test_NonAdminCannotRevokeRole` -- prank as nonOwner, expect revert

**Section 3: renounceRole (TEST-01)**
- `test_AccountCanRenounceOwnRole` -- grant TEST_ROLE to account1, prank as account1, renounceRole(TEST_ROLE, account1), verify role removed
- `test_RenounceRoleRequiresCallerConfirmation` -- prank as account1, renounceRole(TEST_ROLE, account2) should revert with AccessControlExtendedCanOnlyRenounceForSelf
- `test_RenounceDefaultAdminRoleAllowed` -- grant DEFAULT_ADMIN_ROLE to account1, account1 can renounce it (OZ standard behavior, safe because owner has implicit admin)
- `test_RenounceRoleClearsData` -- grant TEST_ROLE with data to account1, account1 renounces, verify data is cleared

**Section 4: hasRole (TEST-01)**
- `test_HasRoleReturnsTrueForGrantedRole` -- straightforward positive case
- `test_HasRoleReturnsFalseForNonGrantedRole` -- straightforward negative case
- `test_HasRoleReturnsFalseAfterRevocation` -- grant then revoke, verify false

**Section 5: Enumeration - forward lookup (TEST-01)**
- `test_GetRoleMemberReturnsCorrectAddress` -- grant role to multiple accounts, verify getRoleMember returns correct address by index
- `test_GetRoleMemberCountReflectsGrantsAndRevocations` -- grant 3, revoke 1, count should be 2
- `test_GetRoleMemberRevertsOnOutOfBounds` -- expect revert when index >= count

**Section 6: Reverse lookup (TEST-01)**
- `test_RolesOfReturnsAllRoles` -- grant multiple roles to account1, verify rolesOf returns all of them
- `test_RolesOfReturnsEmptyForNoRoles` -- account with no roles returns empty array
- `test_RolesOfUpdatesAfterRevocation` -- grant 2 roles, revoke 1, verify only 1 remains
- `test_RolesOfReturnsConsistentWithHasRole` -- for each role in rolesOf output, hasRole returns true

**Section 7: Auxiliary data - setRoleData / getRoleData / grantRoleWithData (TEST-01)**
- `test_SetRoleDataStoresData` -- setRoleData then getRoleData returns same bytes
- `test_SetRoleDataEmitsEvent` -- vm.expectEmit for RoleDataChanged(role, account, data)
- `test_SetRoleDataAllowedWithoutRole` -- per CONTEXT.md, setRoleData does NOT revert if account lacks role (pre-configuration pattern)
- `test_SetRoleDataRequiresAdminAuthority` -- prank as nonOwner, expect revert with AccessControlExtendedUnauthorized
- `test_GetRoleDataReturnsEmptyForNoData` -- returns empty bytes for account with no data
- `test_GrantRoleWithDataStoresRoleAndData` -- grantRoleWithData, verify hasRole and getRoleData
- `test_GrantRoleWithDataEmitsBothEvents` -- vm.expectEmit for RoleGranted AND RoleDataChanged
- `test_GrantRoleWithDataUpdateDataIfRoleAlreadyHeld` -- grant role first, then grantRoleWithData with data -> should update data, emit RoleDataChanged (no duplicate RoleGranted since role already held)
- `test_GrantRoleWithDataEmptyDataNoDataEvent` -- grantRoleWithData with empty data -> only RoleGranted, no RoleDataChanged

**Section 8: Data cleanup on revoke (TEST-01)**
- `test_RevokeRoleClearsAssociatedData` -- grantRoleWithData, revoke, getRoleData returns empty
- `test_RevokeRoleEmitsRoleDataChangedWhenDataExists` -- vm.expectEmit for RoleDataChanged(role, account, "") when data existed
- `test_RevokeRoleNoDataEventWhenNoDataExists` -- revoke role that had no data -> no RoleDataChanged event

**Section 9: Role admin hierarchy (TEST-01)**
- `test_DefaultAdminRoleIsDefaultAdmin` -- getRoleAdmin(TEST_ROLE) returns DEFAULT_ADMIN_ROLE
- `test_SetRoleAdminChangesAdmin` -- use internal _setRoleAdmin (need to expose via mock) to set custom admin, verify getRoleAdmin returns new admin, verify RoleAdminChanged event
- `test_CustomAdminCanGrantItsRole` -- set ANOTHER_ROLE as admin for TEST_ROLE, grant ANOTHER_ROLE to account1, account1 can grant TEST_ROLE
- `test_DefaultAdminCanAlwaysGrantRegardlessOfCustomAdmin` -- even with custom admin set, DEFAULT_ADMIN_ROLE holder (and owner) can still grant (per locked decision: root admin for all roles)

For `_setRoleAdmin` testing, add a public wrapper in the mock: `function setRoleAdmin(bytes32 role, bytes32 adminRole) public { _setRoleAdmin(role, adminRole); }`
